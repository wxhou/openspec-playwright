# 项目规范
- 动手前读 `openspec/config.yaml`（技术栈、结构、约定、约束等），无内容则忽略

## 代码质量
- 动手前先思考：列出假设，逐条验证。复杂任务多种解释则都列出，更简单方案则提出
- 每步有可验证的退出条件：多步骤任务先列计划，循环验证直到成功。lint 失败时优先运行 `npm run lint:fix`
- lint + typecheck 通过才算成功。检测主语言：\`.py\` → ruff+mypy、\`.ts\`/\`.tsx\` → ESLint+tsc、\`.go\` → gofmt+vet
- 只写被要求的东西：不加"灵活"、"可配置"、单次使用的抽象。200行能50行完成则重写
- 精准改动：只改必要的，改完清理自己造成的垃圾。匹配现有风格，不改进无关代码

## 禁止非通用性改动
- 禁止假设外部数据有效 → 必须校验类型/范围/null
- 断言用通用规则，不用具体值（除非明确要求）
- 禁止魔法数字 → 用常量或枚举，注释说明原因
- 禁止隐式成功假设 → 异步/外部操作必须处理失败情况
- 禁止响应结构假设 → 先校验返回结构再访问深层属性
- 禁止资源泄漏假设 → 文件/连接/cursor 等使用后必须释放

## 上下文管理
- 超过 500 行文件分次读取或编辑前重新读取完整内容
- 超过 10 条消息后，编辑任何文件前强制重新读取
- 上下文压缩恢复后：git status → 确认改动范围与设计一致 → 验证 lint+typecheck → 继续

## 工具限制
- 搜索要全：Grep 搜内容 + Glob 搜文件名，两者缺一不可。跳过 node_modules/vendor/__pycache__
- 禁止脚本改文件：用 Read/Edit/Write，格式化工具（prettier/ruff fmt）除外
- 重命名时覆盖调用、类型、字符串、import、barrel file、测试 mock
- 禁止推送：除非用户明确要求，否则不执行 git push

## 大规模任务
- 200+ 行修改或架构变更（新增服务/API 契约/数据模型重构）必须走 OpenSpec 工作流（/opsx:propose），禁止直接修改

## 临时文件管理
- 所有工具（Chrome DevTools MCP、截图、日志、heapdump 等）产生的非源码临时文件必须放项目根目录 `tmp/` 下
- 平铺，不分子目录，文件名含时间戳（避免覆盖）
- `tmp/` 中超过 24 小时的文件可以在 commit 前删除

## 其他
- 始终用中文回复用户
- `npx openspec --help` 查看 OpenSpec 可用命令

## 安全规范
- 密钥与 `.env` 文件不入版本控制
- 示例代码用占位符（如 `YOUR_API_KEY`）不用真实凭据
- 调试日志不打印凭据/密钥/token

OpenSpec Playwright?

专为 OpenSpec 规范驱动开发设计，AI 全流程自动化，测试不再成为交付瓶颈。 Designed for spec-driven development with OpenSpec. AI-powered automation throughout, tests are no longer a delivery bottleneck.

规范驱动 Spec-Driven
规范驱动测试生成 Spec-Driven Test Generation
读取 OpenSpec 规范文档，自动提取路由、表单、验证规则，生成对应 Playwright 测试用例。不再需要手动编写和维护测试。 Reads OpenSpec specs, auto-extracts routes, forms, and validation rules, generates Playwright tests. No more manual test writing and maintenance.
规范即文档 Specs as Docs
规范变更，测试即更新 Spec Changes = Test Updates
测试与 OpenSpec 规范严格对齐。验证通过意味着规范得到实现，规范变更自动驱动测试更新。 Tests strictly aligned with OpenSpec specs. Passing verification means spec is implemented. Spec changes auto-drive test updates.
自愈机制 Self-Healing
Healer 自愈 Healer Auto-Fix
选择器失效时 AI 自动诊断并修复。区分测试 Bug 和 App Bug，三阶段修复流程：分类 → 修复 → 升级。 AI auto-diagnoses and fixes selector failures. Distinguishes Test Bug vs App Bug with 3-phase repair: Triage → Fix → Escalate.

真实浏览器探索 Real Browser Exploration
通过浏览器工具遍历每个路由（Playwright MCP / openspec-pw explore），snapshot + @ref 精确定位元素。Token 高效，状态持久，可视化操作。 Traverse routes via a browser tool (Playwright MCP or openspec-pw explore), snapshot + @ref for precise element targeting. Token-efficient, persistent state, visual operations.

完整测试报告 Complete Test Reports
自动生成 Markdown 报告，包含测试结果、截图、失败分类（App Bug / Test Bug / RAFT）。可追溯、可分享。 Auto-generated Markdown reports with results, screenshots, failure classification (App Bug / Test Bug / RAFT). Traceable and shareable.

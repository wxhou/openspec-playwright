# OpenSpec E2E Verification

Run Playwright E2E verification against the OpenSpec specs for a change. This command reads the specs generated by `/opsx:propose` and uses Playwright's three-agent pipeline to validate the implementation.

## Prerequisites

Before running, ensure:
1. Playwright is installed: `npx playwright install`
2. Playwright agents are initialized: `npx playwright init-agents --loop=claude`
3. Playwright MCP is configured in your Claude Code settings
4. The dev server is running (or will be started automatically)

## Workflow

### Step 1: Identify the Change

Ask the user which change to verify, or default to the most recent change in `openspec/changes/`.

### Step 2: Read OpenSpec Specs

Read all files from `openspec/changes/<change-name>/specs/*.md`. These are the requirements that tests must cover.

### Step 3: Generate Test Plan (Planner Agent)

Create `openspec/changes/<change-name>/specs/playwright/test-plan.md` by:
- Extracting all functional requirements from the specs
- Listing each requirement as a test case
- Including expected behaviors and acceptance criteria

### Step 4: Generate Tests (Generator Agent)

Use the Playwright Generator agent to create tests in `tests/playwright/<change-name>.spec.ts`:
- Follow the format of `tests/playwright/seed.spec.ts` if it exists
- Generate tests covering all items in the test plan
- Verify selectors and assertions are valid

### Step 5: Run Tests with Auto-Heal (Healer Agent)

Execute the tests using Playwright:
- Run `npx playwright test tests/playwright/<change-name>.spec.ts`
- If tests fail, use Playwright MCP tools to analyze the current UI state
- Auto-heal broken selectors by finding equivalent elements
- Re-run tests until all pass or guardrails trigger

### Step 6: Report Results

Generate a report at `openspec/reports/playwright-e2e-<change-name>.md` containing:
- Number of tests run / passed / failed
- Any auto-heals performed
- Final pass/fail status
- Recommendations if tests failed

## Output Format

```
## E2E Verify Report: <change-name>

**Status:** ✅ PASS / ❌ FAIL
**Tests:** X/Y passed
**Attempts:** N

### Details
[Auto-heal log]

### Recommendation
[If failed: what needs to be fixed]
```
